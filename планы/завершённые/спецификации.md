## Что было сделано

Написаны и организованы все спецификации проекта перед началом backend-разработки.

### Созданные спеки

| Файл / Папка | Содержание |
|---|---|
| `spec/global_spec.md` | Общее описание проекта, роли, scope, языки |
| `spec/design_spec.md` | Дизайн-система: цвета, шрифты, компоненты, логотип |
| `spec/deployment_spec.md` | Деплой, CI/CD, Vercel, Neon, домен, чеклист запуска |
| `spec/featurespec/` | Спеки функций F-01 — F-13 (каждая в отдельном файле) |
| `spec/tech_spec/` | Стек, архитектура, БД, API, аутентификация, безопасность |
| `spec/functional_map/` | Пользовательские сценарии: клиент и администратор |

### Организация документации

Все три основных спека разбиты на папки с отдельными файлами по разделам:
- `featurespec/` — 13 файлов (F-01 — F-13) + overview + index
- `tech_spec/` — 11 файлов (стек, БД, API, ...) + overview + index
- `functional_map/` — 7 файлов (клиент, трекинг, отмена, ...) + overview + index
