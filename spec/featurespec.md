## F-01 — Просмотр меню

**Назначение:** Клиент видит все доступные позиции ресторана с фото, описанием и ценами.

**Данные, которые видит пользователь:**
- Список роллов с фото
- Название и описание каждого ролла
- Цена (в евро, €)
- Пометка "Скидка" или перечёркнутая старая цена, если есть акция
- Пометка "Нет в наличии" если позиция временно недоступна

**Действия пользователя:**
- Листает меню
- Нажимает на ролл — видит увеличенное фото и полное описание
- Нажимает "Добавить в корзину" с карточки ролла

**Исключительные случаи:**
- Ролл помечен "Нет в наличии" → кнопка "Добавить" неактивна, нельзя добавить в корзину
- Меню пустое (нет ни одной позиции) → показывается сообщение "Меню временно недоступно"

---

## F-02 — Корзина

**Назначение:** Клиент собирает список роллов перед оформлением заказа.

**Данные, которые видит пользователь:**
- Список добавленных позиций
- Количество каждой позиции
- Цена за единицу и итого по позиции
- Общая сумма заказа
- Кнопка "Оформить заказ"

**Действия пользователя:**
- Увеличить или уменьшить количество позиции
- Удалить позицию из корзины
- Перейти к оформлению заказа

**Исключительные случаи:**
- Корзина пустая → кнопка "Оформить заказ" неактивна
- Пользователь уменьшил количество до 0 → позиция автоматически удаляется из корзины
- Ролл стал "Нет в наличии" пока пользователь смотрел меню → при переходе к оформлению показывается предупреждение "Один или несколько роллов больше недоступны"

## F-03 — Оформление заказа

**Назначение:** Клиент указывает свои данные и способ получения, заказ сохраняется на сервере и поступает администратору.

**Данные, которые видит пользователь:**
- Форма с полями: Имя, Фамилия, Телефон
- Выбор способа получения: Доставка / Самовывоз
- Поле "Адрес" — появляется только при выборе доставки
- Итоговая стоимость заказа
- Стоимость доставки (если выбрана доставка)
- Кнопка "Подтвердить заказ"

**Действия пользователя:**
- Заполняет форму
- Выбирает способ получения
- Вводит адрес (если доставка)
- Нажимает "Подтвердить заказ"

**Порядок оформления:**
1. Клиент заполняет форму и нажимает "Подтвердить заказ"
2. Клиент переходит на экран оплаты (F-08)
3. После успешной оплаты — заказ создаётся на сервере и поступает администратору
4. Клиент видит экран подтверждения с номером заказа

> Заказ создаётся ТОЛЬКО после подтверждения оплаты. Если оплата не прошла — заказа нет.

**Исключительные случаи:**
- Не заполнено обязательное поле (Имя, Фамилия, Телефон) → форма не отправляется, выделяется незаполненное поле
- Выбрана доставка но не указан адрес → форма не отправляется
- Номер телефона в неверном формате → показывается ошибка под полем
- Оплата не прошла → заказ НЕ создаётся, клиент остаётся на экране оплаты с сообщением об ошибке
- Сервер недоступен → показывается сообщение "Не удалось оформить заказ. Попробуйте ещё раз."

## F-04 — Расчёт стоимости доставки

**Назначение:** Система автоматически рассчитывает стоимость доставки по адресу клиента на основе расстояния от ресторана.

**Данные, которые видит пользователь:**
- Поле ввода адреса
- Расстояние от ресторана (км) — после ввода адреса
- Стоимость доставки (€) — рассчитывается автоматически
- Таблица тарифов на странице (чтобы клиент знал заранее)

**Действия пользователя:**
- Вводит адрес доставки
- Видит рассчитанную стоимость — до подтверждения заказа

**Исключительные случаи:**
- Адрес не найден в базе геокодирования → "Адрес не найден. Проверьте написание."
- Адрес находится дальше 5 км → "Этот адрес вне зоны доставки. Выберите самовывоз."
- Расчёт занял слишком долго → показывается индикатор загрузки, затем результат или ошибка

**Бизнес-правило — тариф доставки:**

| Расстояние от ресторана | Стоимость доставки |
|---|---|
| 0 — 3 км | €2.50 (минимум) |
| 3.0 — 3.5 км | €3.50 |
| 3.5 — 4.0 км | €4.50 |
| 4.0 — 4.5 км | €5.50 |
| 4.5 — 5.0 км | €6.50 |
| Больше 5 км | Доставка недоступна |

- Зона доставки: радиус 5 км от ресторана (Ле-Ман)

## F-05 — Отмена заказа (3 минуты)

**Назначение:** Клиент может отменить заказ самостоятельно в течение 3 минут после оформления, без звонка в ресторан.

**Данные, которые видит пользователь:**
- Экран подтверждения заказа с таймером обратного отсчёта (3:00 → 0:00)
- Кнопка "Отменить заказ" — активна пока таймер не вышел
- После истечения 3 минут: кнопка исчезает или становится неактивной
- При успешной отмене: сообщение "Ваш заказ отменён"

**Действия пользователя:**
- Нажимает "Отменить заказ" в течение 3 минут
- Подтверждает отмену (диалог: "Вы уверены?")

**Исключительные случаи:**
- Прошло больше 3 минут → кнопка неактивна, отмена невозможна, сообщение "Время для отмены истекло. Свяжитесь с нами по телефону."
- Клиент нажал "Отменить" но сервер не ответил → показывается ошибка, заказ НЕ считается отменённым, предлагается попробовать ещё раз
- Клиент нажал "Отменить" и уже истекло время на сервере (но таймер ещё показывал секунды) → сервер возвращает отказ, клиент видит "Время для отмены истекло"
- Клиент отменил оплаченный заказ в течение 3 минут → система автоматически инициирует возврат денег через Stripe. Клиент видит сообщение: "Заказ отменён. Деньги вернутся на карту в течение 5–10 рабочих дней."

## F-06 — Регистрация и вход

**Назначение:** Клиент может создать аккаунт чтобы получать SMS об акциях. Регистрация необязательна — заказать можно без неё.

**Данные, которые видит пользователь:**
- Форма регистрации: Имя, Фамилия, Номер телефона, Пароль
- Галочка "Согласен(а) получать SMS об акциях и скидках" — по умолчанию снята, клиент ставит сам
- Форма входа: Номер телефона, Пароль
- Кнопка "Войти" / "Зарегистрироваться"
- Ссылка "Заказать без регистрации"

> По закону Франции (RGPD) нельзя отправлять SMS без явного согласия клиента. Галочка обязательна — SMS рассылаются только тем у кого она стоит.

**Действия пользователя:**
- Регистрируется (заполняет форму)
- Входит в аккаунт
- Оформляет заказ — данные формы подтягиваются автоматически если вошёл

**Исключительные случаи:**
- Номер телефона уже зарегистрирован → "Этот номер уже используется. Войдите или восстановите доступ."
- Неверный пароль при входе → "Неверный номер телефона или пароль."
- После 5 неверных попыток входа → временная блокировка на 15 минут
- Клиент забыл пароль → нажимает "Забыли пароль?" → вводит номер телефона → получает SMS с кодом (6 цифр, действует 10 минут) → вводит код → создаёт новый пароль

## F-07 — Отслеживание статуса заказа

**Назначение:** Клиент может видеть на каком этапе находится его заказ после оформления.

**Данные, которые видит пользователь:**
- Номер заказа
- Текущий статус: Новый → Готовится → В доставке → Выполнен
- Время оформления заказа
- Ожидаемое время доставки: 30 минут + 5 минут за каждый км от ресторана
- Карта с местоположением курьера — появляется когда статус стал "В доставке" (см. F-13)

**Действия пользователя:**
- Открывает страницу отслеживания по ссылке из подтверждения заказа
- Или заходит в свой аккаунт и видит историю заказов

**Исключительные случаи:**
- Клиент ввёл неверный номер заказа → "Заказ не найден"
- Заказ отменён → статус показывается как "Отменён"
- Сервер недоступен → "Не удалось загрузить статус заказа. Попробуйте позже."

## F-08 — Онлайн-оплата картой

**Назначение:** Клиент оплачивает заказ банковской картой при оформлении, без наличных.

**Данные, которые видит пользователь:**
- Итоговая сумма к оплате (заказ + доставка), в евро (€)
- Кнопки быстрой оплаты: **Apple Pay**, **Google Pay** — показываются если браузер поддерживает
- Форма оплаты картой: номер карты, срок действия, CVV (Visa / Mastercard)
- Статус платежа: "Оплата обрабатывается..." → "Оплата прошла" / "Оплата не прошла"

**Действия пользователя:**
- Нажимает Apple Pay или Google Pay — оплачивает в одно касание
- Или вводит данные карты вручную и нажимает "Оплатить"
- При 3DS — проходит подтверждение в банковском приложении

**Исключительные случаи:**
- Недостаточно средств на карте → "Платёж отклонён. Недостаточно средств."
- Карта заблокирована → "Платёж отклонён. Обратитесь в банк."
- Истёк срок действия карты → ошибка при вводе на форме
- Ошибка на стороне платёжного провайдера → "Платёж не удался. Попробуйте ещё раз или выберите другую карту."
- Клиент закрыл страницу во время 3DS → заказ НЕ создаётся, деньги не списываются
- Apple Pay / Google Pay недоступны в браузере клиента → показывается только форма карты

**Технические детали:**
- Платёжный провайдер: **Stripe**
- Валюта: только EUR (€)
- Поддерживаемые методы: Visa, Mastercard, Apple Pay, Google Pay
- Данные карты НЕ хранятся на сервере ресторана — всё через Stripe
- Комиссия Stripe: 1.5% + 0.25€ с каждой транзакции (за счёт ресторана)
- **Требование:** перед подключением Stripe необходима регистрация бизнеса во Франции (например, auto-entrepreneur)

## F-09 — SMS-уведомления об акциях

**Назначение:** Зарегистрированные клиенты получают SMS когда владелец запускает акцию или скидку.

**Данные, которые видит пользователь (клиент):**
- SMS на телефон: текст акции, срок действия, ссылка на сайт (опционально)

**Данные, которые видит пользователь (администратор):**
- Форма в панели: текст SMS, кнопка "Разослать всем"
- Статус рассылки: "Отправлено X из Y"

**Действия пользователя:**
- Клиент: получает SMS, переходит на сайт по ссылке
- Администратор: создаёт текст акции, запускает рассылку

**Исключительные случаи:**
- Клиент отписался от SMS → его номер не включается в рассылку
- Ошибка доставки SMS → система повторяет попытку, логирует неудачи
- Администратор отправил рассылку случайно → нет отмены (SMS уже ушли), нужно подтверждение перед отправкой

**Технические детали:**
- SMS отправляются через внешний веб-сервис по API (провайдер: Twilio или Brevo — TBD)
- Администратор управляет рассылкой через панель на сайте, без стороннего интерфейса

## F-10 — Панель администратора: управление меню

**Назначение:** Владелец может добавлять, редактировать и скрывать позиции меню без помощи разработчика.

**Данные, которые видит пользователь (администратор):**
- Список всех позиций меню
- Статус каждой позиции: активна / скрыта
- Форма редактирования: название, описание, цена, фото, наличие скидки

**Действия администратора:**
- Добавить новую позицию
- Изменить название, описание, цену, фото
- Поставить скидку (старая цена → новая цена)
- Скрыть позицию (убрать из меню для клиентов, но сохранить в базе)
- Показать обратно скрытую позицию

> Физическое удаление позиций запрещено — только скрытие (is_visible = false). Это защищает историю заказов: старые заказы всегда показывают правильный состав даже если позиция больше не продаётся.

**Исключительные случаи:**
- Не заполнено обязательное поле (название, цена) → позиция не сохраняется
- Загружено фото слишком большого размера → "Фото слишком большое. Максимум 5 МБ."
- Администратор скрыл позицию → она исчезает из меню для клиентов мгновенно, но остаётся в истории заказов

## F-11 — Панель администратора: управление заказами

**Назначение:** Владелец видит все входящие заказы и меняет их статус в процессе работы.

**Данные, которые видит пользователь (администратор):**
- Список заказов: новые вверху
- По каждому заказу: номер, время, имя клиента, телефон, состав, способ получения, адрес (если доставка), сумма
- Текущий статус заказа
- Кнопки смены статуса

**Действия администратора:**
- Принять заказ → статус "Готовится"
- Передать курьеру → статус "В доставке"
- Закрыть заказ → статус "Выполнен"
- Отменить заказ вручную — только администратор, в любое время, независимо от 3-минутного правила
- Позвонить клиенту — нажать на номер телефона

**Исключительные случаи:**
- Пришёл новый заказ → панель автоматически обновляется каждые 30 секунд, новый заказ появляется вверху списка с визуальным выделением и звуковым сигналом
- Администратор не в сети → заказ сохраняется, появится при следующем обновлении панели
- Попытка изменить статус заказа который уже "Выполнен" → действие заблокировано

## F-13 — GPS-отслеживание курьера

**Назначение:** Клиент видит на карте где находится курьер в реальном времени пока идёт доставка.

> Это сложная функция. Реализуется без мобильного приложения: курьер открывает специальную страницу на своём телефоне, браузер передаёт GPS-координаты на сервер, клиент видит карту.

**Как работает для курьера:**
- Администратор нажимает "Передать курьеру" → заказ получает статус "В доставке"
- Курьер получает ссылку (например, через SMS или WhatsApp от администратора): `oumiroll.fr/courier/track?order=123`
- Курьер открывает ссылку на телефоне → браузер просит разрешение на GPS → курьер разрешает
- Страница автоматически отправляет координаты на сервер каждые 10 секунд пока заказ не выполнен

**Данные, которые видит клиент:**
- Карта (Google Maps) с маркером курьера
- Маркер ресторана и маркер адреса клиента
- Карта обновляется каждые 10 секунд
- Карта появляется только когда статус "В доставке", исчезает когда статус "Выполнен"

**Исключительные случаи:**
- Курьер не открыл ссылку или не дал разрешение на GPS → карта не показывается, клиент видит только статус "В доставке"
- Курьер потерял интернет → последняя известная позиция остаётся на карте, обновление прекращается
- Заказ выполнен → карта скрывается, показывается сообщение "Заказ доставлен"

## F-12 — Система бонусов

**Назначение:** Администратор может включать и выключать бонусы для клиентов в любой момент. Когда бонус активен, клиенты автоматически получают его при оформлении заказа.

**Текущие бонусы:**

| Условие | Бонус |
|---|---|
| Заказ от 2 порций роллов | Бесплатная доставка |
| Заказ от 2 порций роллов | +1 порция роллов бесплатно (администратор выбирает какая) |

**Данные, которые видит клиент:**
- Если бонус активен — баннер на сайте: "Закажи 2 порции и получи бесплатную доставку + 1 порцию в подарок!"
- В корзине при выполнении условия: строка "Бонус применён — доставка бесплатно" и "Подарок: [название ролла]"
- Итоговая сумма уже учитывает бонус

**Данные, которые видит администратор:**
- В панели: список бонусов с переключателем ВКЛ / ВЫКЛ
- Выбор ролла который даётся в подарок

**Действия администратора:**
- Включить или выключить бонус в любой момент
- Выбрать ролл-подарок из списка меню

**Исключительные случаи:**
- Бонус выключен → баннер не показывается, скидка не применяется
- Ролл-подарок скрыт из меню → бонус не активируется, администратору показывается предупреждение
- Клиент убрал из корзины один ролл и условие (2 порции) больше не выполняется → бонус автоматически снимается, стоимость пересчитывается
