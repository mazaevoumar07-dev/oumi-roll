# Rate Limiting

> Ограничения запросов защищают эндпоинты которые стоят денег или могут быть использованы для атаки.
> Реализуется через Next.js middleware на Vercel.

---

## Таблица лимитов

| Эндпоинт | Лимит | Ключ | Причина |
|---|---|---|---|
| `POST /api/delivery/calculate` | 10 запросов / минута | IP | Каждый запрос = платный вызов Google Maps Geocoding API |
| `POST /api/auth/forgot-password` | 3 запроса / 15 минут | номер телефона | Каждый запрос = платная SMS через Twilio; защита от спама |
| `POST /api/auth/login` | 10 запросов / минута | IP | Защита от перебора паролей (в дополнение к блокировке по БД) |
| `POST /api/auth/register` | 5 запросов / минута | IP | Защита от массовой регистрации фейковых аккаунтов |

---

## Ответ при превышении

```
HTTP 429 Too Many Requests
Retry-After: <секунды до сброса>
```

---

## Двойная защита для входа

Для `POST /api/auth/login` работают **два независимых механизма**:

1. **Rate limiting** (middleware) — 10 запросов / минута с одного IP → `429`
2. **Блокировка по БД** (таблица `login_attempts`) — 5 неверных попыток с одного телефона → блокировка на 15 минут

Это разные уровни: rate limiting защищает от быстрых атак с одного IP, блокировка по БД — от медленного перебора с разных IP на один аккаунт.
