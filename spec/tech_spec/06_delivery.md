# Расчёт стоимости доставки

> Связанная функция: F-04. Эндпоинт: `POST /api/delivery/calculate`.

---

## Алгоритм расчёта

1. Клиент вводит адрес
2. Backend отправляет адрес в **Google Maps Geocoding API** → получает координаты (lat, lng)
3. Backend считает расстояние от ресторана до клиента: **по прямой × 1.3** (поправка на дороги)
4. Применяет тарифную таблицу

---

## Тарифная таблица

| Расстояние от ресторана | Стоимость доставки |
|---|---|
| 0 — 3 км | €2.50 |
| 3.0 — 3.5 км | €3.50 |
| 3.5 — 4.0 км | €4.50 |
| 4.0 — 4.5 км | €5.50 |
| 4.5 — 5.0 км | €6.50 |
| Больше 5 км | Доставка недоступна |

---

## Координаты ресторана

Задаются в переменных окружения — не в коде:

```
RESTAURANT_LAT=47.9948
RESTAURANT_LNG=0.1985
```

---

## Когда вызывается расчёт

Запрос отправляется **один раз при уходе из поля** (`onBlur`) — не на каждый символ.
Это защищает от лишних вызовов Google Maps Geocoding API (каждый вызов платный).

Если клиент вернулся в поле и изменил адрес — расчёт повторяется при следующем `onBlur`.
Пока расчёт выполняется — поле заблокировано, показывается индикатор загрузки.

---

## Rate limiting

`POST /api/delivery/calculate` — **10 запросов / минута** с одного IP.
Причина: каждый запрос = платный вызов Google Maps Geocoding API.
