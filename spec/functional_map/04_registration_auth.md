# Регистрация, вход и восстановление пароля

> Связанная функция: F-06 (Регистрация и вход)

---

## Главное правило

**Регистрация необязательна.** Клиент может оформить заказ без аккаунта.

Зачем тогда регистрироваться:
- Получать SMS об акциях и скидках
- Видеть историю своих заказов без токена
- При оформлении заказа — форма заполняется автоматически

---

## Регистрация

**Что заполняет клиент:**
- Имя
- Фамилия
- Номер телефона
- Пароль
- Галочка: "Согласен(а) получать SMS об акциях и скидках" *(по умолчанию снята, клиент ставит сам)*

> По закону Франции (RGPD) нельзя отправлять SMS без явного согласия. Галочка обязательна — SMS рассылаются только тем у кого она стоит.

---

## Вход

- Номер телефона + пароль
- Ссылка "Заказать без регистрации" — всегда доступна

---

## Восстановление пароля

1. Клиент нажимает "Забыли пароль?"
2. Вводит номер телефона
3. Получает SMS с кодом (6 цифр, действует 10 минут)
4. Вводит код на сайте
5. Создаёт новый пароль

---

## История заказов

Зарегистрированный клиент может зайти в аккаунт и увидеть все свои прошлые заказы — без токена, просто через сессию.

**Что видит:**
- Список всех заказов: дата, состав, сумма, статус
- Каждый заказ — ссылка на страницу отслеживания `/suivi/[id]`

**Когда полезно:**
- Потерял ссылку на отслеживание текущего заказа — находит его через аккаунт
- Хочет посмотреть что заказывал раньше

**Незарегистрированный клиент** историю не видит — только по токену из письма/экрана подтверждения.

---

## Ситуации

### Номер телефона уже зарегистрирован
- "Этот номер уже используется. Войдите или восстановите доступ."

### Неверный пароль при входе
- "Неверный номер телефона или пароль."

### 5 неверных попыток входа подряд
- Временная блокировка на 15 минут

### Клиент не ставил галочку на SMS
- Он зарегистрирован, но не получает SMS-рассылки
- Акции его не касаются

### Клиент заказывает без регистрации
- Заполняет форму вручную каждый раз
- Не получает SMS об акциях
- Может видеть заказ только по ссылке с токеном
