## Зачем пользователь заходит на сайт

Три ситуации:

1. **Хочет заказать прямо сейчас** — голодный, выбирает и оформляет заказ сразу
2. **Планирует заказ заранее** — смотрит меню, думает что взять, оформляет чуть позже
3. **Просто смотрит** — изучает меню, цены, акции без намерения заказать прямо сейчас

---

## Как пользователь решает свою задачу (основной путь)

1. Заходит на сайт
2. Просматривает меню и выбирает роллы
3. Добавляет нужные позиции в корзину
4. Переходит к оформлению заказа
5. Выбирает способ получения: **доставка** или **самовывоз**
6. Заполняет данные:
   - Имя
   - Фамилия
   - Номер телефона
   - Адрес доставки *(только если выбрана доставка)*
7. Видит стоимость доставки *(рассчитывается по километражу)*
8. Оплачивает картой (Visa / Mastercard / Apple Pay / Google Pay)
9. Получает экран подтверждения с номером заказа и ссылкой для отслеживания

---

## Ситуации которые могут случиться

### Клиент выбрал самовывоз
- Адрес вводить не нужно
- Стоимость доставки не показывается

### Клиент выбрал доставку
- Вводит адрес
- Видит стоимость доставки перед подтверждением заказа

### Клиент хочет узнать стоимость доставки заранее
- На сайте есть таблица тарифов — сколько стоит доставка в зависимости от расстояния

### Ролл помечен "Нет в наличии"
- Кнопка "Добавить в корзину" неактивна
- Ролл виден в меню с пометкой "Нет в наличии"

### Клиент хочет посмотреть историю заказов
- Заходит в аккаунт → видит все прошлые заказы: дата, состав, сумма, статус
- Из истории можно перейти к странице отслеживания любого заказа
- Только для зарегистрированных клиентов

### Клиент хочет отменить заказ
- Клиент может отменить заказ **только в течение 3 минут** после оформления
- Отмена делается самим клиентом через сайт — без звонка
- После 3 минут отмена невозможна — заказ уже в работе
- Почему 3 минуты: за это время ресторан ещё не начинает готовить, потерь нет

### Клиент отменил оплаченный заказ
- Система автоматически инициирует возврат денег через Stripe
- Деньги возвращаются на карту в течение 5–10 рабочих дней

## После оформления заказа — отслеживание (F-07, F-13)

После успешной оплаты клиент получает ссылку: `/suivi/[id]?token=[tracking_token]`

По этой ссылке клиент видит:
- Текущий статус заказа (обновляется автоматически)
- Ожидаемое время доставки
- Карту с курьером — появляется только когда статус "В доставке"

**Статусы:** Новый → Готовится → В доставке → Выполнен

**Ситуации:**
- Клиент зарегистрирован — может видеть историю заказов через аккаунт без токена
- Курьер не дал доступ к GPS — карта не показывается, только статус
- Клиент потерял ссылку и не зарегистрирован — статус не увидит

## Акции и бонусы

### Скидки на роллы
- На некоторые роллы будут скидки
- Какие именно роллы — определяет администратор в панели

### Бонус при заказе 2 порций роллов
- **Бесплатная доставка**
- **+1 порция роллов бесплатно** — администратор выбирает какая порция даётся в подарок
- Если бонус выключен администратором — не применяется и не показывается

## Backend-функции (текущий этап)

> Эти функции реализуются в рамках второго этапа (backend). Спеки написаны в featurespec.md.

### Регистрация и вход (F-06)
- Клиент может заказать **без регистрации** — регистрация необязательна
- Если клиент хочет — может зарегистрироваться (Имя, Фамилия, Телефон, Пароль)
- Вход по номеру телефона и паролю
- **Восстановление пароля:** клиент вводит номер телефона → получает SMS с кодом (6 цифр, 10 минут) → вводит код → создаёт новый пароль

### Отслеживание заказа (F-07)
- Клиент открывает страницу отслеживания по ссылке из подтверждения
- Видит статус, время и карту с курьером

### GPS-отслеживание курьера (F-13)
- Администратор передаёт заказ курьеру → генерируется ссылка для курьера
- Курьер открывает ссылку на телефоне → браузер передаёт GPS-координаты каждые 10 секунд
- Клиент видит карту в реальном времени

### SMS-уведомления о скидках (F-09)
- Зарегистрированные клиенты (с согласием на SMS) получают уведомления когда владелец запускает акцию
- Владелец рассылает уведомление из панели администратора

### Отмена заказа через сайт (F-05)
- Клиент видит таймер — сколько времени осталось для отмены (3 минуты)
- Нажимает кнопку "Отменить заказ" — заказ отменяется без звонка
- После истечения 3 минут кнопка исчезает или становится неактивной
- При отмене оплаченного заказа — автоматический возврат денег через Stripe

### Онлайн-оплата (F-08)
- Оплата картой при оформлении заказа через Stripe
- Поддержка: Visa, Mastercard, Apple Pay, Google Pay
- Заказ создаётся ТОЛЬКО после подтверждения оплаты

## Администратор как отдельный пользователь (F-10, F-11, F-12)

Владелец ресторана работает через отдельную защищённую панель.

**Вход:** страница `/admin/login` — скрыта от клиентов. Логин + пароль. Аккаунт создаётся один раз разработчиком, регистрации нет.

### Что делает администратор:
- **Меню:** добавляет, редактирует, скрывает позиции; ставит скидки
- **Заказы:** принимает заказы, меняет статусы, отменяет вручную в любое время
- **Курьер:** нажимает "Передать курьеру" → отправляет ссылку курьеру (SMS / WhatsApp)
- **Бонусы:** включает / выключает акции, выбирает ролл-подарок
- **SMS-рассылка:** создаёт текст акции и рассылает всем согласившимся клиентам
