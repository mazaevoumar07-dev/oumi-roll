# Global Spec — OUMI ROLL

## Что это за проект

Это сайт для ресторана суши-роллов. Сайт позволяет клиентам смотреть меню и оформлять заказы онлайн, а владельцу ресторана — управлять меню и принимать заказы без лишних звонков.

---

## Кто будет пользоваться сайтом

### Клиент ресторана
- Обычный человек, который хочет заказать суши. Доступные варианты:
  - **Доставка** — оформить заказ онлайн и получить дома
  - **Самовывоз** — оформить заказ онлайн и забрать самому
  - **На месте** — прийти лично и сделать заказ на месте
  - **По телефону** — позвонить и продиктовать заказ
- Открывает сайт с телефона или компьютера
- Хочет быстро найти нужный ролл, добавить в корзину и оформить заказ

### Владелец / администратор ресторана
- Управляет меню: добавляет новые блюда, меняет цены, убирает позиции
- Видит входящие заказы и меняет их статус (новый → готовится → доставляется → выполнен)
- Заходит через отдельную защищённую панель

## Язык сайта

- **Основной язык:** французский — сайт открывается на французском по умолчанию
- **Переключатель языка** в шапке сайта: 🇫🇷 Français / 🇬🇧 English / 🇷🇺 Русский
- Выбранный язык сохраняется в браузере (localStorage) — клиент не выбирает снова при следующем визите
- Все три языка применяются ко всему сайту: меню, формы, сообщения об ошибках, кнопки

---

## Где будет находиться сайт

- В интернете — доступен по домену (например, `oumiroll.fr` или аналогичный)
- Работает на любом устройстве: телефон, планшет, компьютер
- Основная аудитория открывает сайт с мобильного телефона, поэтому мобильная версия — приоритет
- **Географическое расположение бизнеса:** Франция, город Ле-Ман (Le Mans)
- Аудитория сайта — жители Ле-Мана и его окрестностей

## Зачем нужен этот сайт

| Проблема сейчас | Что решает сайт |
|---|---|
| Нет удобного способа заказать | Клиент выбирает: онлайн, самовывоз, на месте или по телефону |
| Ошибки при приёме заказа по телефону | Онлайн-заказ фиксируется точно, как выбрал клиент |
| Нет удобного способа показать меню | Красивое меню с фото и ценами всегда доступно |

## Scope — что входит и что нет

### Этап 1 — Frontend (завершён)
- Статичный сайт без сервера и без API
- Меню, корзина, форма заказа — всё на стороне клиента

### Этап 2 — Backend (текущий этап)
- Сервер, база данных, API
- Реальная обработка и сохранение заказов
- Онлайн-оплата картой (Stripe)
- Регистрация и вход клиентов
- SMS-уведомления об акциях
- Панель администратора (меню и заказы)
- Отслеживание статуса заказа
- Расчёт стоимости доставки по геокодингу

### Не входит (вне scope)
- Мобильное приложение
- Несколько ресторанов / филиалов
