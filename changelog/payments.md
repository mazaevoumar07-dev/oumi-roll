# Changelog — Оплата (Stripe)

Все изменения в flow оплаты, интеграции Stripe, webhook.

---

## [Этап 1] 2026-05-06 — 2026-05-18

### Добавлено
- Страница оплаты `/paiement` — mock-версия с Stripe Elements (F-08)
- Данные формы (имя, адрес, корзина) сохраняются в `metadata` Payment Intent

---

## 2026-05-31 — 2026-06-05

### Добавлено
- `POST /api/payment/create-intent` — создание Payment Intent со всеми данными заказа в metadata
- `POST /api/payment/webhook` — обработка события `payment_intent.succeeded`: создание заказа в БД, запуск SMS-уведомления
- Страница подтверждения `/confirmation` — показывает детали заказа после успешной оплаты
- Email-чек через Stripe (встроенная функция) — без кастомной реализации

### Исправлено
- Удалён `config bodyParser` из webhook — конфликтовал с App Router и ломал верификацию подписи Stripe

---

## 2026-06-18 — 2026-06-19

### Добавлено
- Предупреждение об изменении цены — если цена товара изменилась с момента добавления в корзину, клиент видит уведомление перед оплатой (вариант B: предупреждение, не блокировка)
- `POST /api/admin/orders` — создание заказов без онлайн-оплаты (наличные на месте, оплата при получении)
