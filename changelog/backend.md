# Changelog — Backend / API

Все изменения в API-маршрутах, базе данных, серверной логике.

---

## 2026-05-31 — 2026-06-05

### Добавлено
- **Supabase** — настройка клиента (server и browser), middleware, схема БД, миграции
- `GET /api/menu` — список всех позиций меню
- `GET /api/menu/:id` — одна позиция меню
- `POST /api/delivery/calculate` — расчёт стоимости и доступности доставки (Google Maps + радиус)
- `GET /api/orders/:id` — информация о заказе
- `POST /api/orders/:id/cancel` — отмена заказа клиентом (в течение 3 минут после создания)
- `GET /api/users/me/orders` — история заказов авторизованного пользователя
- Авторизация переведена с mock на **Supabase Auth** (OTP по телефону)
- Полный flow оплаты: создание Payment Intent → webhook → создание заказа в БД

### Исправлено
- Противоречия в коде, выявленные при анализе проекта

---

## 2026-06-06 — 2026-06-07

### Добавлено
- `DELETE /api/users/me` — удаление аккаунта и персональных данных (RGPD ст. 17)

### Исправлено
- Убран CRLF-символ из `NEXT_PUBLIC_APP_URL` через `.trim()` в `next.config.ts` — приводил к ошибкам запросов на Vercel
- Исключены `/api/*` из глобального middleware — каждый маршрут проверяет авторизацию сам
- Удалён устаревший `config bodyParser` из webhook-обработчика (не нужен в App Router)

---

## 2026-06-18 — 2026-06-19

### Добавлено
- `POST /api/admin/orders` — ручное создание заказов администратором (по телефону или на месте)
- **F-14:** `GET /api/working-hours` и `GET /api/working-hours/status` — часы работы и статус паузы
- Логика скрытия заказов в панели администратора в течение 3-минутного окна отмены — заказ не отображается, пока клиент может его отменить
