# F-10 — Панель администратора: управление меню

**Этап:** Backend

**Назначение:** Владелец может добавлять, редактировать и скрывать позиции меню без помощи разработчика.

---

## Что видит администратор

- Список всех позиций меню (включая скрытые)
- Статус каждой позиции: активна / скрыта
- Форма редактирования: название, описание, цена, фото, скидка

---

## Два разных состояния позиции

| Состояние | Поле в БД | Что означает | Кнопка в панели |
|---|---|---|---|
| **Нет в наличии** | `is_available = false` | Ролл виден в меню но добавить нельзя — временно закончился | "Нет в наличии / В наличии" |
| **Скрыто** | `is_visible = false` | Ролл полностью убран из меню — клиент его не видит | "Скрыть / Показать" |

> Физическое удаление позиций запрещено. Это защищает историю заказов: старые заказы всегда показывают правильный состав.

---

## Действия администратора

- Добавить новую позицию (название, описание, цена, фото)
- Изменить название, описание, цену, фото существующей позиции
- Поставить скидку (старая цена → новая цена)
- Отметить "Нет в наличии" — ролл виден в меню с пометкой, кнопка "Добавить" неактивна
- Скрыть позицию — исчезает из меню для клиентов мгновенно, остаётся в базе
- Показать обратно скрытую позицию или вернуть в наличие

---

## Загрузка фото

- Файл загружается через форму → сохраняется в **Supabase Storage** (бакет `menu-photos`) → в БД пишется только URL
- Поддерживаемые форматы: **JPG, PNG, WebP**
- Максимальный размер: **5 МБ**

---

## Исключительные случаи

- Не заполнено обязательное поле (название, цена) → позиция не сохраняется
- Загружено фото слишком большого размера → "Фото слишком большое. Максимум 5 МБ."
- Администратор скрыл позицию → она исчезает из меню для клиентов мгновенно, но остаётся в истории заказов
