# F-11 — Панель администратора: управление заказами

**Этап:** Backend

**Назначение:** Владелец видит все входящие заказы и меняет их статус в процессе работы.

---

## Что видит администратор

- Список заказов: новые вверху
- По каждому заказу: номер, время, имя клиента, телефон, состав, способ получения, адрес (если доставка), сумма
- Текущий статус заказа
- Кнопки смены статуса

> **Заказы появляются в панели только через 3 минуты после оформления** — пока клиент может отменить самостоятельно, администратор заказа не видит. Если клиент отменил в течение 3 минут — заказ не появится в панели вовсе.

---

## Действия администратора

- Принять заказ → статус "Готовится"
- Передать курьеру → статус "В доставке"
- Закрыть заказ → статус "Выполнен"
- Отменить заказ вручную — в любое время, независимо от 3-минутного правила
- Позвонить клиенту — нажать на номер телефона
- **Кнопка-переключатель «Приём заказов: Активен / На паузе»** — временно блокирует новые заказы от клиентов при перегрузке кухни (подробнее — F-14)

---

## Возврат денег при ручной отмене

Если администратор отменяет **оплаченный** заказ:
- Система автоматически инициирует возврат через Stripe Refunds API
- Клиент получает деньги обратно на карту в течение 5–10 рабочих дней
- В панели администратора показывается подтверждение: "Заказ отменён, возврат инициирован"

---

## Исключительные случаи

- Пришёл новый заказ → панель обновляется каждые 30 секунд, новый заказ появляется вверху с визуальным выделением и **звуковым сигналом**
- Администратор не в сети → заказ сохраняется, появится при следующем обновлении панели
- Попытка изменить статус заказа "Выполнен" → действие заблокировано

---

## Ручное создание заказа (телефон / на месте)

Администратор может создать заказ вручную — для клиентов, которые позвонили или пришли лично.

### Поля формы

| Поле | Обязательно | Примечание |
|---|---|---|
| Источник | да | «По телефону» / «На месте» |
| Имя клиента | да | |
| Телефон клиента | нет | Обязателен только для доставки |
| Позиции заказа | да | Выбор из текущего меню |
| Способ получения | да | Доставка / Самовывоз / На месте |
| Адрес | только для доставки | |
| Способ оплаты | да | Наличные / Карта на месте |
| Примечание | нет | Комментарий к заказу |

### Отличия от онлайн-заказа

- **Stripe не используется** — оплата наличными или терминалом на месте; возврат тоже вручную
- **3-минутное окно отмены не применяется** — заказ сразу видит администратор
- **SMS клиенту не отправляется** — администратор общается с клиентом напрямую
- **Статус после создания** — сразу «Готовится» (без этапа «Новый»)

### Исключительные случаи

- Позиция из меню «нет в наличии» → не показывается в форме создания заказа
- Не заполнены обязательные поля → заказ не сохраняется

---

## Ролл закончился после оплаты клиента

**Правило:** если хотя бы один ролл из оплаченного заказа закончился — **полная отмена заказа**. Частичных возвратов нет.

### Порядок действий администратора:

1. Отметить ролл как «нет в наличии» в `/admin/menu` — чтобы следующие клиенты его не видели
2. Нажать «Отменить заказ» в панели — Stripe автоматически возвращает полную сумму клиенту
3. Система автоматически отправляет SMS клиенту на номер из заказа

### SMS клиенту (отправляется автоматически при отмене по этой причине):

> *Votre commande a été annulée car un article n'est plus disponible. Vous serez remboursé sous 5 à 10 jours ouvrés. Vous pouvez passer une nouvelle commande sur notre site — l'article indisponible n'apparaîtra plus.*

### Что видит клиент:
- Возврат полной суммы на карту в течение 5–10 рабочих дней (через Stripe)
- SMS с объяснением и приглашением заказать снова

### Что требует реализации:
- При отмене заказа администратором — система должна автоматически отправлять SMS клиенту через Twilio на номер телефона из заказа
- Сейчас кнопка «Отменить» только инициирует возврат через Stripe, SMS не отправляется
