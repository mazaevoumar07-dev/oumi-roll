# F-04 — Расчёт стоимости доставки

**Этап:** Backend

**Назначение:** Система автоматически рассчитывает стоимость доставки по адресу клиента на основе расстояния от ресторана.

---

## Что видит пользователь

- Поле ввода адреса
- Расстояние от ресторана (км) — после ввода адреса
- Стоимость доставки (€) — рассчитывается автоматически
- Таблица тарифов на странице (чтобы клиент знал заранее)

---

## Действия пользователя

- Вводит адрес доставки
- Видит рассчитанную стоимость — до подтверждения заказа

---

## Когда вызывается расчёт

Запрос `POST /api/delivery/calculate` отправляется **один раз при уходе из поля** (`onBlur`) — не на каждый символ. Это защищает от лишних вызовов Google Maps API (каждый вызов платный).

Если клиент вернулся в поле и изменил адрес — расчёт повторяется при следующем `onBlur`.
Пока расчёт выполняется — поле заблокировано, показывается индикатор загрузки.

---

## Тарифная таблица

| Расстояние от ресторана | Стоимость доставки |
|---|---|
| 0 — 3 км | €2.50 |
| 3.0 — 3.5 км | €3.50 |
| 3.5 — 4.0 км | €4.50 |
| 4.0 — 4.5 км | €5.50 |
| 4.5 — 5.0 км | €6.50 |
| Больше 5 км | Доставка недоступна |

Зона доставки: радиус 5 км от ресторана (Ле-Ман).

---

## Исключительные случаи

- Адрес не найден → "Адрес не найден. Проверьте написание."
- Адрес дальше 5 км → "Этот адрес вне зоны доставки. Выберите самовывоз."
- Расчёт занял слишком долго → показывается индикатор загрузки, затем результат или ошибка
