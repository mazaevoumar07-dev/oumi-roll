# F-07 — Отслеживание статуса заказа

**Этап:** Backend

**Назначение:** Клиент может видеть на каком этапе находится его заказ после оформления.

---

## Что видит пользователь

- Номер заказа
- Текущий статус: Новый → Готовится → В доставке → Выполнен
- Время оформления заказа
- Ожидаемое время доставки: 30 минут + 5 минут за каждый км от ресторана
- Карта с местоположением курьера — появляется когда статус "В доставке" (см. F-13)

---

## Действия пользователя

- Открывает страницу отслеживания по ссылке из подтверждения заказа: `/suivi/[id]?token=[tracking_token]`
- Или заходит в свой аккаунт и открывает нужный заказ из истории

> `tracking_token` генерируется сервером при создании заказа (UUID v4), хранится в таблице `orders`. Страница `/suivi/[id]` требует либо валидный `?token=`, либо авторизованную сессию владельца этого заказа — без одного из двух возвращает 403.

---

## История заказов (для зарегистрированных клиентов)

**Что видит клиент на странице истории:**
- Список всех своих заказов: дата, состав, итоговая сумма, статус
- По каждому заказу — ссылка "Отследить" → переходит на `/suivi/[id]`

**Незарегистрированный клиент** историю не видит — только по ссылке с токеном из экрана подтверждения.

**API:** `GET /api/users/me/orders`

---

## Исключительные случаи

- Клиент ввёл неверный токен → "Заказ не найден"
- Заказ отменён → статус показывается как "Отменён"
- Сервер недоступен → "Не удалось загрузить статус заказа. Попробуйте позже."
