# F-03 — Оформление заказа

**Этап:** Backend

**Назначение:** Клиент указывает свои данные и способ получения, заказ сохраняется на сервере и поступает администратору.

---

## Что видит пользователь

- Форма с полями: Имя, Фамилия, Телефон
- Выбор способа получения: Доставка / Самовывоз
- Поле "Адрес" — появляется только при выборе доставки
- Итоговая стоимость заказа
- Стоимость доставки (если выбрана доставка)
- Кнопка "Подтвердить заказ"

---

## Действия пользователя

- Заполняет форму
- Выбирает способ получения
- Вводит адрес (если доставка)
- Нажимает "Подтвердить заказ"

---

## Порядок оформления

1. Клиент заполняет форму и нажимает "Подтвердить заказ"
2. Сервер создаёт Stripe Payment Intent — данные формы (имя, фамилия, телефон, адрес, состав корзины, тип доставки, итоговая сумма) передаются в `metadata` Payment Intent
3. Сервер **независимо** проверяет условия бонуса и рассчитывает итоговую сумму — клиентской сумме не доверяет
4. Клиент переходит на экран оплаты (F-08)
5. После успешной оплаты — Stripe отправляет webhook на сервер → сервер читает `metadata` из Payment Intent → создаёт заказ с уникальным `tracking_token`
6. Клиент видит экран подтверждения с номером заказа и ссылкой для отслеживания: `/suivi/[id]?token=[tracking_token]`

> Заказ создаётся ТОЛЬКО после подтверждения оплаты через Stripe webhook. Если оплата не прошла — заказа нет.
> Данные формы хранятся в metadata Stripe Payment Intent до подтверждения — не в нашей БД.

---

## Исключительные случаи

- Не заполнено обязательное поле (Имя, Фамилия, Телефон) → форма не отправляется, выделяется незаполненное поле
- Выбрана доставка но не указан адрес → форма не отправляется
- Номер телефона в неверном формате → ошибка под полем
- Оплата не прошла → заказ НЕ создаётся, клиент остаётся на экране оплаты с сообщением об ошибке
- Сервер недоступен → "Не удалось оформить заказ. Попробуйте ещё раз."
