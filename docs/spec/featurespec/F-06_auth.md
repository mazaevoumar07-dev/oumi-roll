# F-06 — Регистрация и вход

**Этап:** Backend

**Назначение:** Клиент может создать аккаунт чтобы получать SMS об акциях и видеть историю заказов. Регистрация необязательна — заказать можно без неё.

> Auth реализован через **Supabase Auth**. Никакого ручного JWT/bcrypt.

---

## Что видит пользователь

- Форма регистрации: Имя, Фамилия, Номер телефона, Пароль
- Галочка "Согласен(а) получать SMS об акциях и скидках" — по умолчанию снята, клиент ставит сам
- Форма входа: Номер телефона, Пароль
- Кнопка "Войти" / "Зарегистрироваться"
- Ссылка "Заказать без регистрации"

> По закону Франции (RGPD) нельзя отправлять SMS без явного согласия клиента. Галочка обязательна — SMS рассылаются только тем у кого она стоит.

---

## Действия пользователя

- Регистрируется (заполняет форму)
- Входит в аккаунт
- Выходит из аккаунта (кнопка "Выйти")
- При оформлении заказа — данные формы подтягиваются автоматически

---

## Поток регистрации (Supabase Phone Auth)

1. Клиент заполняет форму (Имя, Фамилия, Телефон, Пароль)
2. Фронтенд вызывает `supabase.auth.signUp({ phone, password })`
3. Supabase отправляет OTP-код через Twilio на указанный телефон
4. Клиент вводит 6-значный код подтверждения
5. Фронтенд вызывает `supabase.auth.verifyOtp({ phone, token, type: 'sms' })`
6. Supabase создаёт пользователя в `auth.users`, наш сервер создаёт строку в `public.users` (через Supabase trigger или API route)

---

## Поток входа

1. Клиент вводит телефон + пароль
2. Фронтенд вызывает `supabase.auth.signInWithPassword({ phone, password })`
3. Supabase возвращает сессию, `@supabase/ssr` сохраняет её в `httpOnly cookie`

---

## Восстановление пароля

1. Клиент нажимает "Забыли пароль?"
2. Вводит номер телефона
3. Фронтенд вызывает `supabase.auth.resetPasswordForEmail` (или phone OTP flow)
4. Получает SMS с кодом → вводит → создаёт новый пароль
5. Supabase обновляет пароль — дополнительного кода не требует

---

## История заказов

Зарегистрированный и вошедший клиент может открыть страницу своих заказов.

**Что видит:**
- Список всех своих заказов: дата, состав, итоговая сумма, статус

**API:** `GET /api/users/me/orders` — возвращает только заказы текущего клиента (проверяется через Supabase сессию).

---

## Исключительные случаи

- Номер телефона уже зарегистрирован → Supabase возвращает ошибку `"User already registered"` → показать "Этот номер уже используется. Войдите или восстановите доступ."
- Неверный пароль при входе → Supabase возвращает ошибку → показать "Неверный номер телефона или пароль."
- Слишком много попыток входа → Supabase автоматически блокирует (встроенный rate limiting)
- OTP-код просрочен или неверный → Supabase возвращает ошибку → "Неверный код. Запросите новый."
