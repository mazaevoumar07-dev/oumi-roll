# F-05 — Отмена заказа (3 минуты)

**Этап:** Backend

**Назначение:** Клиент может отменить заказ самостоятельно в течение 3 минут после оформления, без звонка в ресторан.

---

## Что видит пользователь

- Экран подтверждения заказа с таймером обратного отсчёта (3:00 → 0:00)
- Кнопка "Отменить заказ" — активна пока таймер не вышел
- После истечения 3 минут: кнопка исчезает или становится неактивной
- При успешной отмене: сообщение "Ваш заказ отменён"

---

## Действия пользователя

1. Нажимает "Отменить заказ" в течение 3 минут
2. Подтверждает отмену (диалог: "Вы уверены?")
3. Заказ отменяется

---

## Возврат денег

Если клиент отменил **оплаченный** заказ в течение 3 минут:
- Система автоматически инициирует возврат через Stripe Refunds API
- Клиент видит: "Заказ отменён. Деньги вернутся на карту в течение 5–10 рабочих дней."

---

## Статус заказа не влияет на право отмены

**Правило:** 3-минутное окно абсолютно. Даже если администратор уже перевёл заказ в статус «Готовится» — клиент всё равно может отменить, пока не истекли 3 минуты.

Отмена невозможна только если заказ уже в статусе `completed` (выполнен) или `cancelled` (уже отменён). Статусы `preparing` и `in_delivery` отмену не блокируют.

**Последствие для администратора:** заказ появляется в панели только через 3 минуты после создания. Если клиент отменил в течение 3 минут — администратор этот заказ вообще не увидит.

---

## Исключительные случаи

- Прошло больше 3 минут → кнопка неактивна, сообщение "Время для отмены истекло. Свяжитесь с нами по телефону."
- Клиент нажал "Отменить" но сервер не ответил → ошибка, заказ НЕ считается отменённым, предложение попробовать ещё раз
- Таймер ещё показывал секунды, но на сервере время уже вышло → сервер возвращает отказ, клиент видит "Время для отмены истекло"

---

## Сбой возврата Stripe

Если при вызове Stripe Refunds API возникла ошибка (сетевой сбой, карта заблокирована, и т.д.):

**Статус заказа:** помечается `cancelled` независимо от ошибки возврата — отмена и возврат денег не связаны жёстко.

**Что видит клиент:**
> "Votre commande a été annulée, mais un problème est survenu lors du remboursement. Veuillez nous contacter par téléphone."

**Что делает сервер:**
- Устанавливает флаг `refund_failed = true` в таблице `orders`
- Логирует ошибку с `charge_id` и текстом от Stripe

**Что делает администратор:**
- В панели заказ отображается с пометкой "Возврат не выполнен"
- Открывает Stripe Dashboard, находит платёж по `charge_id`, делает возврат вручную
