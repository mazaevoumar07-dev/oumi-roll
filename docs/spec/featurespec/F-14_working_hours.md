# F-14 — Часы работы, предзаказ и пауза приёма заказов

**Этап:** Backend

**Назначение:** Система знает когда ресторан открыт, позволяет клиентам оформить предзаказ на нужное время, и даёт администратору кнопку экстренной паузы при перегрузке кухни.

---

## Часы работы

| День | Открытие | Закрытие | Последний слот заказа |
|---|---|---|---|
| Понедельник – Суббота | 11:00 | 22:00 | 21:30 |
| Воскресенье | 12:00 | 21:00 | 20:30 |

> Последний слот — за 30 минут до закрытия: кухне нужно время на приготовление.

---

## Временны́е слоты

- Шаг: **30 минут**
- Пример слотов в понедельник: 11:00, 11:30, 12:00, … , 21:00, 21:30
- Пример слотов в воскресенье: 12:00, 12:30, … , 20:00, 20:30

---

## Что видит клиент при оформлении заказа

### Ресторан открыт прямо сейчас

Клиент видит выпадающий список со слотами:

1. **«Dès que possible (~45 min)»** — первый вариант, выбран по умолчанию
2. Ближайший слот через 30 мин от текущего времени
3. Все следующие слоты до конца рабочего дня

### Ресторан закрыт (ночь, выходной или пауза)

Клиент видит баннер:

> *«Nous sommes actuellement fermés. Vous pouvez passer une commande pour notre prochaine ouverture.»*
> («Мы сейчас закрыты. Вы можете оформить предзаказ на следующее время открытия.»)

Выпадающий список показывает слоты **следующего рабочего дня**, начиная с первого слота открытия. Оплата списывается сразу — ресторан получит заказ в момент открытия.

---

## Пауза приёма заказов (перегрузка кухни)

Администратор может временно остановить новые заказы прямо из панели, не меняя часы работы.

### Что видит администратор

- В `/admin` — кнопка-переключатель: **«Прием заказов: Активен / На паузе»**
- При нажатии: статус меняется мгновенно, кнопка меняет цвет (зелёный → красный)

### Что происходит для клиентов во время паузы

- Меню доступно для просмотра
- При попытке оформить заказ — баннер вместо формы:
  > *«Nous ne prenons pas de nouvelles commandes pour le moment. Veuillez réessayer dans quelques minutes.»*
  > («Мы временно не принимаем новые заказы. Пожалуйста, попробуйте через несколько минут.»)
- Кнопка "Passer la commande" недоступна

### Поведение паузы

- Пауза **не отменяет** уже принятые заказы — они остаются в панели и продолжают готовиться
- Пауза не связана с часами работы — можно поставить на паузу и в рабочее время
- При закрытии браузера администратора пауза **сохраняется** в БД — снять её нужно вручную

## Техническая реализация

### Таблица `restaurant_settings` (новая миграция)

```sql
CREATE TABLE restaurant_settings (
  key   TEXT PRIMARY KEY,
  value TEXT NOT NULL
);

INSERT INTO restaurant_settings (key, value) VALUES ('orders_paused', 'false');
```

### Новые API-маршруты

| Метод | Маршрут | Описание | Доступ |
|---|---|---|---|
| GET | /api/status | Статус: открыт ли ресторан + пауза | Все |
| PATCH | /api/admin/settings/pause | Включить / выключить паузу | Admin |

**GET /api/status** возвращает:
```json
{
  "open": true,
  "paused": false,
  "nextOpenAt": null
}
```
Если `open: false` — `nextOpenAt` содержит ISO-дату следующего открытия.

### Слоты на клиенте

Слоты генерируются на клиенте на основе часов работы (константы). Сервер при создании Payment Intent проверяет, что выбранный слот попадает в рабочее время и ресторан не на паузе.

### Хранение выбранного слота

Поле `delivery_time` (тип `TIMESTAMPTZ`) добавляется в таблицу `orders`. Значение `NULL` означает «как можно скорее».

---

## Исключительные случаи

- Клиент выбрал слот, но пока он оплачивал — ресторан закрылся или поставил паузу → сервер отклоняет Payment Intent с ошибкой, клиент видит объяснение
- Клиент выбрал последний слот дня (21:30), но оплата заняла время и сейчас уже 21:35 → тот же отказ на сервере
