# После оформления заказа

> Связанные функции: F-03 (Оформление заказа), F-05 (Отмена), F-07 (История заказов)

---

## Что происходит после успешной оплаты

1. Stripe подтверждает оплату → сервер создаёт заказ
2. Stripe автоматически отправляет чек об оплате на email клиента (через `receipt_email`)
3. Клиент видит экран подтверждения:
   - Номер заказа
   - "Чек об оплате отправлен на [email]"
   - Таймер на отмену (3 минуты) — см. [03_cancellation_refund.md](03_cancellation_refund.md)

> Публичной страницы отслеживания нет. Анонимный клиент видит только экран подтверждения и получает чек на email.

---

## Статусы заказа

```
Новый → Готовится → В доставке → Выполнен
```

| Статус | Кто меняет |
|---|---|
| Новый | Создаётся автоматически при оплате |
| Готовится | Администратор |
| В доставке | Администратор |
| Выполнен | Администратор |
| Отменён | Клиент (3 мин) или администратор |

Статусы видит только администратор в панели управления заказами. Зарегистрированный клиент видит статус через историю заказов в аккаунте.

---

## История заказов (для зарегистрированных клиентов)

- Клиент заходит в аккаунт → раздел "Мои заказы"
- Видит список всех своих заказов: дата, состав, сумма, статус
- **API:** `GET /api/users/me/orders` (требует Client JWT)

---

## Ситуации

### Клиент не получил чек на email
- Проверить папку "Спам"
- Stripe повторно не отправляет — клиент может обратиться в ресторан по телефону с номером заказа

### Клиент хочет узнать статус заказа
- Если зарегистрирован — заходит в аккаунт → "Мои заказы"
- Если не зарегистрирован — связывается с рестораном по телефону
