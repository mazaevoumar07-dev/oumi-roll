# Администратор как отдельный пользователь

> Связанные функции: F-10 (Панель: меню), F-11 (Панель: заказы), F-09 (SMS-рассылка), F-12 (Бонусы), F-13 (GPS-курьер)

---

## Кто это

Владелец ресторана или менеджер.

---

## Вход в панель администратора

- Страница входа: `/admin/login` — не отображается в навигации, скрыта от обычных клиентов
- Логин: email + пароль
- Аккаунт создаётся **один раз разработчиком** — формы регистрации нет
- Только один администратор — владелец ресторана
- После входа — JWT-токен хранится в httpOnly cookie, сессия действует 7 дней

**Ситуации:**
- Неверный пароль → "Неверный логин или пароль."
- Попытка открыть `/admin/*` без входа → перенаправляется на `/admin/login`
- Сессия истекла → перенаправляется на `/admin/login`

---

## Что умеет администратор

### 1. Управление меню

**Цель:** обновлять меню без помощи разработчика.

**Действия:**
- Добавить новую позицию (название, описание, цена, фото)
- Изменить название, описание, цену, фото существующей позиции
- Поставить скидку (старая цена → новая цена)
- Отметить «Нет в наличии» — ролл виден в меню с пометкой, кнопка «Добавить» неактивна (`is_available = false`)
- Вернуть в наличие — пометка исчезает, кнопка снова активна
- Скрыть позицию — она полностью исчезает из меню для клиентов, но остаётся в базе (`is_visible = false`)
- Показать обратно скрытую позицию

> Физическое удаление позиций запрещено — только скрытие. Это защищает историю заказов: старые заказы всегда показывают правильный состав.

**Ситуации:**
- Не заполнено обязательное поле → позиция не сохраняется
- Загружено фото больше 5 МБ → "Фото слишком большое. Максимум 5 МБ."

---

### 2. Управление заказами

**Цель:** видеть входящие заказы и вести их через этапы.

**Что видит:**
- Список заказов: новые вверху
- По каждому: номер, время, имя, телефон, состав, способ получения, адрес, сумма
- Текущий статус

**Действия:**
- Принять заказ → статус "Готовится"
- Передать курьеру → статус "В доставке" + генерируется ссылка для курьера
- Закрыть заказ → статус "Выполнен"
- Отменить заказ вручную — в любое время, без ограничения 3 минут; если заказ был оплачен — система автоматически инициирует возврат через Stripe, клиент получает деньги за 5–10 рабочих дней
- Позвонить клиенту — нажать на номер телефона

**Ситуации:**
- Новый заказ → панель обновляется каждые 30 секунд, новый заказ появляется вверху с визуальным выделением и звуковым сигналом
- Администратор не в сети → заказ сохраняется, появится при следующем обновлении
- Попытка изменить статус заказа "Выполнен" → действие заблокировано

---

### 3. Передача заказа курьеру (GPS)

**Действия:**
1. Нажимает "Передать курьеру" → сервер генерирует уникальную ссылку для курьера
2. Отправляет ссылку курьеру (SMS / WhatsApp)
3. Курьер открывает ссылку → браузер передаёт GPS-координаты

**Ситуации:**
- Курьер не открыл ссылку → карта у клиента не показывается, только статус "В доставке"

---

### 4. Управление бонусами

**Действия:**
- Включить или выключить бонус "2 порции → бесплатная доставка + подарок"
- Выбрать ролл-подарок из списка меню

**Ситуации:**
- Ролл-подарок скрыт из меню (`is_visible = false`) → бонус не активируется, администратору показывается предупреждение
- Ролл-подарок не в наличии (`is_available = false`) → бонус не активируется, предупреждение «Ролл-подарок закончился, выберите другой»

---

### 5. SMS-рассылка об акциях

**Действия:**
1. Создаёт текст акции
2. Нажимает "Разослать всем"
3. Подтверждает отправку (чтобы не отправить случайно)
4. Видит статус: "Отправлено X из Y"

**Ситуации:**
- Отмена рассылки невозможна — SMS уже ушли
- Поэтому перед отправкой обязательное подтверждение
- Клиент без галочки "Согласен на SMS" → не получит рассылку
