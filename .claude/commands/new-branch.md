# New Branch

Создать ветку от актуального `main` перед началом задачи.

Если аргумент передан (например `/new-branch feat/stripe-integration`) — использовать его как имя ветки. Иначе спросить у пользователя тип и название.

## Типы веток

| Тип | Когда использовать |
|---|---|
| `feat/` | Новая функциональность |
| `fix/` | Исправление бага |
| `docs/` | Изменения только в документации |
| `chore/` | Служебные изменения: конфиги, зависимости |

Примеры: `feat/stripe-integration`, `fix/jwt-cookie`, `docs/tech-spec-security`

## Шаги

1. Переключиться на `main` и подтянуть последние изменения:

```bash
git checkout main && git pull origin main
```

2. Создать новую ветку:

```bash
git checkout -b <тип>/<название>
```

3. Сообщить пользователю на какой ветке теперь находимся и что можно начинать работу.
